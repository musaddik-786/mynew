3️⃣ Real retrieval layers engineers implement
Typical production retrieval stack:
Layer 1 — Query processing
Fix vague queries.
Example:
User:
Year built
System rewrites:
What year was the property built?
Property construction year
This improves recall.
⸻
Layer 2 — Multi-query retrieval
Generate query variations.
Example:
construction year
year built
year constructed
property age
Search all.
Merge results.
⸻
Layer 3 — Hybrid retrieval (very common)
Combine:
Vector search + keyword search.
Because embeddings miss exact terms.
Example:
Policy number:
ABC-123-XYZ
Vector search may miss.
Keyword search finds instantly.
Real companies almost always use hybrid search.
⸻
Layer 4 — MMR ranking
Remove duplicate chunks.
Ensure diversity.
⸻
Layer 5 — Metadata filtering
Example:
If question about:
claims
Filter:
section = claims
Avoid irrelevant data.
⸻
Layer 6 — Context compression
Instead of giving full chunks:
Extract only relevant sentences.
Reduces token cost.
Improves answers.
⸻

9️⃣ Example production-style retriever code
Install if needed:
pip install langchain langchain-openai langchain-chroma

⸻
Improved retriever example:
from langchain.retrievers.multi_query import MultiQueryRetriever
from langchain_openai import AzureChatOpenAI
# LLM for query expansion
llm = AzureChatOpenAI(
   openai_api_key=os.getenv("AZURE_OPENAI_API_KEY"),
   azure_endpoint=os.getenv("AZURE_OPENAI_ENDPOINT"),
   openai_api_version=os.getenv("AZURE_OPENAI_API_VERSION"),
   deployment_name=os.getenv("AZURE_OPENAI_CHAT_DEPLOYMENT"),
   temperature=0
)
# Base retriever
base_retriever = vectordb.as_retriever(
   search_type="mmr",
   search_kwargs={
       "k":5,
       "fetch_k":20
   }
)
# Multi query retriever
retriever = MultiQueryRetriever.from_llm(
   base_retriever,
   llm
)
results = retriever.invoke(user_query)
for doc in results:
   print(doc.page_content)

  ↓

  ↓
Return better context
This is a real production retrieval pattern.
